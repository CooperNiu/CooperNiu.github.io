中级软考笔记

数据仓库是一个面向主题的、集成的、相对稳定的、反映历史变化的数据集合，用于支持管理决策。

数据仓库的系统结构

  1. 数据源：是数据仓库系统的基础，是整个系统的数据源泉
  2. 数据的存储与管理：是数据仓库的核心
  3. OLAP服务器：对分析需要的数据进行有效集成，按照多维模型予以组织，以便于进行多角度、多层次分析，并发现趋势
  4. 前端工具：包括各种查询、报表、分析、数据挖掘工具以及各种基于数据藏书的应用开发工具。

OSI协议，共7层

  1. 物理层：网线
  2. 数据链路层
  3. 网络层：网络地址翻译为对应的物理地址。如IP协议
  4. 传输层：TCP、UDP
  5. 会话层：RPC
  6. 表示层：JPEG，GIF等数据格式
  7. 应用层：HTTP，Telnet，TFP等对软件提供接口以使程序能够使用网络服务

项目管理

项目立项管理包括以下5个典型环节

  1. 项目建议
  2. 项目可行性分析
  3. 项目审批
  4. 项目招投标
  5. 项目合同谈判与签订

项目建议

又称为：立项申请

项目建议书主要内容：

  1. 项目简介
  2. 项目建设单位概括
  3. 项目建设的必要性
  4. 业务分析
  5. 总体建设方案
  6. 本期项目建设方案
  7. 环保、消防、职业安全
  8. 项目实施进度
  9. 投资估算和资金筹措
  10. 效益与风险分析

项目可行性分析

一般包括：

  1. 投资必要性
  2. 技术可行性
  3. 财务可行性
  4. 组织可行性
  5. 经济可行性
  6. 社会可行性
  7. 风险因素及对策

可行性报告内容：

  1. 项目概况
  2. 项目建设单位概括
  3. 需求分析和项目建设的必要性
  4. 总体建设方案
  5. 本期项目建设方案
  6. 项目招标方案
  7. 环保、消防、职业安全
  8. 项目组织机构和人员培训
  9. 项目实施进度
  10. 投资估算和资金来源
  11. 效益与评价指标分析
  12. 项目风险与项目管理

项目招标

资格预审文件或者招标文件的发售期不得少于5日

资格预审的申请人少于3个的，应当重新招标

一个招标项目只能有一个标底。标底必须保密

履约保证金不得超过中标合同金额的10%

评标委员会由具有高级职称或者同等专业水平的技术、经济等相关领域专家，招标人和招标机构代表等5人及以上单数组成，其中经济、技术等方面专家人数不少于总数的2/3

合同谈判阶段，先谈技术条款，后谈商务条款。

技术谈判主要包括：合同技术附件内容、合同实施技术路线、质量评定标准、采购设备和系统报价及人员投入开发的比重等

商务谈判主要包括：投标价的优惠条件；质量、工期、服务违约罚款；其他。

项目整体管理

包括6个过程（口诀：二制指监变结）

  1. 制定项目章程
  2. 制定项目管理计划
  3. 指导与管理项目工作
  4. 监控项目工作
  5. 实施整体变更控制。审查所有变更请求，批准变更
  6. 结束项目

制定项目章程

制定项目章程是编写一份正式批准项目并授权项目经理在项目活动中使用组织资源的文件的过程

宣告项目的正式启动、项目经理的任命

由高级管理层签发

遵循“谁签发，谁有权修改”的原则

项目章程的作用：

  1. 确定项目经理，规定其权力
  2. 正式确认项目的存在，给项目一个合法的地位
  3. 规定项目的总体目标，如范围、时间、成本和质量等
  4. 通过叙述启动项目的理由，爸项目与执行组织的日常经营运作及战略计划等联系起来。

制定项目章程的输入

  1. 项目工作说明书，包括：业务需要、产品范围描述、战略计划
  2. 商业论证

项目范围确认

项目范围确认的输入

  1. 项目管理计划
  2. 需求文件
  3. 需求跟踪矩阵
  4. 核实的可交付成果
  5. 工作绩效数据

项目范围确认的输出

  1. 验收的可交付结果
  2. 变更请求
  3. 工作绩效信息
  4. 项目文件更新

七种质量管理工具

口诀：因流核帕直控散

  1. 因果图，鱼骨图、石川图，用来追溯问题来源，推导出根本原因，列尽每根鱼骨上的合理可能性
  2. 流程图，显示输入转化为输出的过程中，需要的步骤顺序和可能分支
  3. 核查表，计数图，收集数据的查对清单
  4. 帕累托图，二八法则，ABC分析法，抓重点原因，识别造成绝大多数问题的少数主要原因
  5. 直方图，描述集中趋势、发散程度和统计分布形状
  6. 控制图，判断某一过程处于控制之中还是失控状态，和时间有关系
  7. 散点图，两个变量之间的相关性

缩短工期的方法（必考：赶快加减使改）

  1. 赶工，投入更多资源或者增加工作时间，以缩短关键活动的工期
  2. 快速跟进，并行施工，以缩短关键路径的长度
  3. 加强质量管理，及时发现问题，减少返工
  4. 减小活动范围或降低活动要求
  5. 使用高素质资源或经验丰富的人员
  6. 改进方法或技术，以提高生产效率

团队的5个阶段（必考：形震规发结）

形成阶段：开始形成共同目标，对团队未来有美好期待

震荡阶段：开始执行任务，一般会遇到超出想象的困难，希望被现实打破。个体之间开始争执，互相指责，开始怀疑项目经理的能力

规范阶段：经过一段时间磨合，团队成员之间相互熟悉和了解，矛盾基本解决，项目经理得到团队认可。

发挥阶段：随着相互之间配合默契和对项目经理的信任，成员积极工作，努力实现目标。集体荣誉感非常强，常将团队换成第一称谓，如“我们组，我们部门”等

结束阶段

不管团队目前属于什么阶段，新增加一个人或者减少一个人，都是从形成阶段重新开始。

项目成本管理

成本管理的过程

  1. 制定成本管理计划
  2. 成本估算，编制所需的大致成本
  3. 成本预算，合计各个活动的估算成本，以建立成本基准
  4. 成本控制

成本的类型：

  1. 可变成本
  2. 固定成本
  3. 直接成本
  4. 间接成本
  5. 机会成本：泛指一切做出选择后其中一个最大的损失
  6. 沉没成本：由于过去的决策已经发生了的，不能由现在或将来的任何决策改变的成本。对现有决策而言是不可控成本，会很大程度影响人们的行为方式与决策，在投资决策时应该排除沉没成本的干扰。

应急储备是为了应对已经接受的已识别风险，属于成本基准的一部分

管理储备是为了管理控制的目的而特别流出的项目预算，应对不可预见的工作，“未知-未知”风险。不包括在成本基准中，但属于项目总预算的一部分，需要高层管理者审批

成本管理计划的输入：

口诀：二项事组

  1. 项目管理计划
  2. 项目章程
  3. 事业环境因素
  4. 组织过程资产

成本管理计划的工具和技术（专分会）

  1. 专家判断
  2. 分析技术
  3. 会议

成本管理计划的输出：

  1. 精确等级
  2. 测量单位
  3. 组织程序链接
  4. 控制临界值
  5. 挣值规则
  6. 报告格式
  7. 过程说明
  8. 其他细节

活动是为了完成工作包所需要进行的工作，有可能一个工作包包括多个活动。

保密性、可用性、完整性是信息安全最为关注的三个属性。

项目成功受到4个方面的约束：范围、时间、成本、质量
